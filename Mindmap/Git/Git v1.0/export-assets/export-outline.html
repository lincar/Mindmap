      • git remote
        查看远程和为远程服务器命名<remote-name >

        • 查看当前的远程仓库
        • git remove -v
          查看远程仓库的详细信息
        • git remove add [remote-name ] [url]
          添加远程仓库,名字自己创建
        • git remove set-url <remote-name> <url>
        • git remove show [remote-name]
          查看某个远程仓库的详细信息
          告诉git push 缺少推送的分支，哪些分支没有同步到本地，哪些分支在服务器上已经被删除，以及git pull 将合并哪些分支
        • git remote rename <A> <B>
        • git remote rm <A>
      • git fetch
        Fetch 命令只会将远程的数据拉倒本地仓库，并不会自动合并到当前工作分支
        • git fetch [remote-name]
        • git fetch [remote-name] [branch-name]
      • git pull
        拉取并且自动合并
      • git push
        push的remote-name需要用remote查看和创建。fetch下来后也是一个无法修改的分支（和原始的origin/master一样），要修改的话，要么pull合并，要么自己在这里再创建一个分支出来。
        • git push [remote-name ] [local-branch-name]:[remote-branch-new-name]
          把本地的分支推送到远程分支并且合并，如果远程分支不存在，就创建一个
        • git push [remote-name] [branch-name]
          这里的branch-name 会匹配远程服务器的分支名是否有相同，如果存在，合并，没有创建一个新的分支
        • git push -u [remote-name] [branch-name]
          用-u制定一个默认的remote-name（主机）
        • git push [remote-name] : [branch-name]
          删除远程仓库的分支（通过本地分支名字留空后）
        • git push [remote-name ] --tags
          推送所有的标签

    • Branches
      • command
        • git branch
          CRUD分支
          *表示当前所在的分支
          • git branch -v
            查看每一个分支最后提交的commit
          • git branch -d <branch-name >
            删除分支。D是强制删除
          • git branch --merged
            查看已经合并的分支，没有*就代表已经没有价值了（合并后留下来的产物）
          • git branch --no-merged
            没有合并的分支
        • git checkout
          转移HEAD并且在本地创建分支或跟踪分支
          • git checkout -b <branch-name>
            创建分支并且将HEAD转到分支上
            相当于：
            git branch <branch-name >
            git checkout <branch-name >

          • git checkout -b <branch-name> [remote-name/remote-branch-nane]
            跟踪远程分支，从远程分支中checkout出来的本地分支
          • git checkout --track [remote-name/remote-branch-nane]
            功能同上，本地创建分支简化名字的生成（默认和remote-branch-nane一样）
          • git checkout --orphan <branch-name >
            创建该分支将会成为一个独立的分支，没有以前的历史。
        • git merge <branch-name >
          在master上合并分支，如果合并和分支都在同一个文件夹上的话，需要手工修改
        • git rebase master
          Rebase（后退到共同祖先然后进行重演）后需要master快进合并一下
          {
          git checkout rico
          git rebase master
          git checkout master
          git merge rico
          }
          衍合的目的是为了得到清爽的一条线历史，所以得到后还是要快进合并。

          一旦有分支在开发（或者分支对象发布到公告仓库），远程仓库就不要对该分支进行衍合操作。
